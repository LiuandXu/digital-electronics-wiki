## 问题1: Mermaid "Syntax error in text" (系统性错误)

### 严重等级: 严重

### 问题描述

所有包含 Mermaid 流程图的页面（共 32 个页面，39 个图表）均出现 "Syntax error in text" 错误。Mermaid 图表虽然生成了 SVG，但 SVG 内部包含大号错误文字 "Syntax error in text" 和 "mermaid version 11.16.0"，导致图表不可读。

### 根因分析

问题出在 `docs/javascripts/mermaid.js` 的重新初始化逻辑：

1. 页面加载时 `DOMContentLoaded` 事件触发 `initMermaid()`，Mermaid 首次渲染成功
2. `MutationObserver` 监听 `data-md-color-scheme` 属性变化，在页面加载过程中该属性可能被 MkDocs Material 修改
3. `MutationObserver` 触发 `initMermaid()` 再次执行
4. 此时 `.mermaid` div 的内容已是 SVG（不再是原始 Mermaid 源码）
5. `el.removeAttribute('data-processed')` 移除已处理标记
6. `mermaid.run()` 尝试将 SVG 内容作为 Mermaid 源码重新解析 → 解析失败 → "Syntax error in text"

## 问题3: Mermaid 标签含 `<` 符号

### 严重等级: 中等

### 问题描述

2 个页面的 Mermaid 节点标签中包含 `<` 符号。`<` 在 Mermaid 中可能被解析为 HTML 标签起始符，即使在使用双引号 `["..."]` 的标签中也可能导致解析错误。

## 非问题 (排除项)

以下问题经源码分析确认为误报：

### 1. MathJax 块级公式不匹配 (ch05/5-5)

源码分析检测到 `ch05/5-5.md` 中 `\[` 出现 17 次，`\]` 出现 11 次。经手动检查，多出的 6 个 `\[` 实际上是 LaTeX 间距命令 `\\[4pt]` 中的 `\[`，用于在 `aligned` 环境中添加行间距，并非公式定界符。MathJax 渲染正常。

### 2. 空章节

源码分析检测到约 110 处"标题后直接跟下一个标题"的情况。经检查，这些均为父级标题下直接跟子级标题（如 `## 二、TTL集成门电路的电路结构` 后跟 `### 1. 总体结构`），属于正常的文档结构，非问题。
